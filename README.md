# Prompt Robustness Testing

A simulated AI evaluation portfolio project focused on testing how AI responses behave under ambiguity, formatting changes, conflicting instructions and edge cases.

## Project Purpose

This project demonstrates a structured approach to prompt robustness testing.

The goal is to evaluate whether an AI system can follow instructions consistently when the prompt becomes unclear, conflicting, overly specific or intentionally difficult to interpret.

## What This Project Includes

* Test prompts designed around different failure modes
* Expected behavior for each prompt type
* Observed failure patterns
* Notes on instruction following and response consistency
* Summary of findings and recommendations

## Testing Focus

The project evaluates AI behavior across the following areas:

* Instruction following
* Ambiguity handling
* Formatting consistency
* Conflicting instructions
* Constraint adherence
* Over-answering and under-answering
* Hallucination risk
* Safety and refusal behavior
* Edge-case interpretation

## Prompt Categories

| Category                 | Purpose                                                                  |
| ------------------------ | ------------------------------------------------------------------------ |
| Ambiguous Prompt         | Tests whether the AI asks for clarification or makes careful assumptions |
| Conflicting Instructions | Tests instruction hierarchy and constraint handling                      |
| Format Constraint        | Tests whether the AI follows exact output structure                      |
| Edge Case Prompt         | Tests unusual or incomplete user requests                                |
| Prompt Injection Style   | Tests whether the AI resists irrelevant or conflicting instructions      |
| Over-Specification       | Tests whether too many constraints reduce answer quality                 |

## Evaluation Method

Each prompt is reviewed using:

* Expected behavior
* Observed behavior
* Failure type
* Severity
* Improvement notes

## Disclaimer

This is a simulated portfolio case study.
No confidential data, private customer data or proprietary evaluation material is used.
